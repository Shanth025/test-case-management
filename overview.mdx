The Test Case Management (TCM) tool is an end-to-end platform that streamlines testing operations across the software development lifecycle. It gives teams complete visibility into the following areas:

* Test plan status and pending approvals.
* Automation coverage across features, including automated, manual, and disabled test cases.
* Assignee progress and test case execution status.
* KPI trends and bottlenecks across testing cycles.

From initial test planning and stakeholder approvals through execution and final sign-off, the TCM tool keeps every role informed through intelligent notifications and comprehensive analytics.

## Who should use this tool

The TCM tool is designed for the following four primary roles:

* **QA leads:** Plan test cycles, build test plans, submit for review, and manage sign-off.
* **Developers:** Create and update test cases, track automation status, and monitor coverage contribution.
* **Project managers:** Monitor approval status, filter features by workflow state, and track testing milestones.
* **Stakeholders:** Review KPI dashboards, assess automation coverage, and confirm release readiness.

## Key features

The following capabilities are available across the platform:

* **Feature management:** Create features, assign approvers, link test cases, and track each feature through its workflow lifecycle from draft to sign-off.

* **Test case management:** Create, update, search, filter, and delete test cases. The tool supports bulk operations, advanced filtering, full version history, revert, and admin-only restore.

* **Import from Google Sheets:** Bulk import test cases from a Google Sheet by mapping row ranges to features. Import history is available for review.

* **Workflow and approvals:** Each feature moves through a defined approval workflow. Test engineers submit test plans for review, approvers receive email notifications, and the workflow advances or reverts based on approval decisions.

* **Notifications:** In-app and email notifications alert users to pending actions. Users can view unread items, mark them as read, and review notification history.

* **Comments and collaboration:** Add threaded comments to features and test cases, mention teammates using an `@mention`, and add emoji reactions.

* **KPI dashboard:** View automation coverage metrics by team, priority, test case type, and automation status (automated, manual, and automatable).

* **User and role management:** Admins add users, update roles, and manage access. The tool supports two roles: `USER` and `ADMIN`.
